## ✅ 已完成的核心修改（COMPLETED）

### 1. 摘要重写（Abstract Rewrite）
**问题**：原摘要超长（约400词）、含公式、结构复杂  
**NC要求**：≤200词，无引用，无公式  
**修改内容**：
- 完全重写为188词精简版
- 移除所有公式（包括 $K(...) \le ...$）
- 移除枚举列表，改为流畅段落
- 保留核心信息：问题、方法、结果、意义
- 强调"open-source code"和"reproducible"

**文件位置**：`main.tex` 第79-81行

## ⚠️ 关键待办事项（CRITICAL TODO - 投稿前必须完成）

### A. 代码仓库发布（URGENT - 硬性要求）

**截止时间**：投稿前  
**NC政策**：代码必须公开可访问，不得"upon request"

**操作步骤**：

#### 步骤1：创建GitHub公开仓库
```bash
# 建议仓库名称
eboc-experiments
# 或
eboc-natcomm2025
```

#### 步骤2：上传必需内容
```
eboc-experiments/
├── README.md              # 安装说明、运行示例、依赖说明
├── requirements.txt       # Python依赖（numpy, matplotlib, etc.）
├── LICENSE                # MIT或其他开源协议
├── experiments/
│   ├── t4_reconstruction.py      # T4厚边界重构实验
│   ├── t5_brudno_convergence.py  # T5 Brudno收敛实验
│   ├── t17_subjective_time.py    # T17主观时间实验
│   └── utils.py                  # 共享工具函数
├── figures/
│   ├── generate_fig1.py   # 生成Fig.1的脚本
│   ├── generate_fig2.py   # 生成Fig.2的脚本
│   └── ...
└── data/                  # （可选）示例输出数据
```

#### 步骤3：Zenodo归档获取DOI
1. 访问 https://zenodo.org/
2. 连接GitHub账号
3. 启用仓库同步
4. 在GitHub创建release（如 v1.0）
5. Zenodo自动生成DOI（格式：10.5281/zenodo.XXXXXX）

#### 步骤4：更新main.tex
替换以下占位符：
```latex
% 第1354行
https://github.com/[repository-name]/eboc-experiments
% 改为 ↓
https://github.com/YOUR-USERNAME/eboc-experiments

% 第1354行
10.5281/zenodo.[pending]
% 改为 ↓
10.5281/zenodo.1234567  % 实际DOI号
```

**预计工作量**：2-4小时（含代码整理、README撰写、测试）
